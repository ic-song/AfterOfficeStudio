# 기획

## 제품 정의

프로젝트가 제공하는 제품이나 서비스를 설명합니다.

## 핵심 사용자 흐름

```text
1. 사용자가 무엇을 시작하는가
2. 어떤 입력이나 행동을 하는가
3. 어떤 결과를 얻는가
4. 왜 다시 사용하거나 비용을 낼 수 있는가
```

## 초기 MVP 범위

- MVP에 반드시 들어갈 기능을 적습니다.
- 수익 검증에 필요한 최소 기능을 우선합니다.

## MVP에서 제외

- 초기 속도를 늦추는 기능을 적습니다.
- 비용, 심사, 외부 연동, 복잡한 운영이 필요한 기능은 신중하게 제외합니다.

## MVP 이후 확장 후보

- 반응이 확인된 뒤 추가할 기능을 적습니다.

## 제품 원칙

- 프로젝트마다 지켜야 할 제품 원칙을 적습니다.
- 수익창출 목표와 연결되지 않는 과한 기능은 피합니다.
