# 팀버러쉬형 모바일 게임 서비스 개발 계획

작성일: 2026-06-13

## 목적

Timber Rush의 성장 루프를 참고해 Android 서비스 출시를 목표로 하는 모바일 게임을 개발한다.

이 계획의 목표는 단순 프로토타입이 아니라, 실제 서비스까지 이어질 수 있도록 개발 순서, 검증 기준, 출시 준비 항목을 정리하는 것이다.

## 기준 기획

- 분석 문서: `50_프로젝트/모바일게임/분석/팀버러쉬-게임분석.md`
- 기획 문서: `50_프로젝트/모바일게임/분석/팀버러쉬형-모바일게임-기획.md`
- 임시 게임명: Rune Grove

## 서비스 목표

| 항목 | 방향 |
| --- | --- |
| 우선 플랫폼 | Android |
| 장르 | 캐주얼 증분형 로그라이트 |
| 세션 길이 | 1런 60~120초 |
| 핵심 루프 | 전투 -> 룬 수집 -> 런 중 선택 성장 -> 영구 강화 -> 재도전 |
| 수익화 | 보상형 광고 중심 검토. 아직 확정 아님 |
| 출시 방식 | Google Play 폐쇄 테스트 후 프로덕션 출시 |

## 개발 단계

### 1단계: 웹 플레이어블 MVP

목표는 재미 검증이다. Unity 이전에 브라우저에서 빠르게 실행 가능한 버전을 만든다.

포함 범위:

- 세로형 모바일 화면
- 플레이어 이동
- 자동 공격
- 몬스터 스폰
- 경험치/레벨업
- 3개 업그레이드 선택
- 60초 런
- 결과 화면
- 룬 획득
- 영구 강화 저장

완료 기준:

- 브라우저에서 실행된다.
- 60초 런이 시작되고 끝난다.
- 몬스터를 처치하면 경험치와 룬을 얻는다.
- 레벨업 선택지가 실제 전투 능력에 반영된다.
- 런 종료 후 영구 강화가 저장된다.

### 2단계: Unity MVP

웹 MVP에서 검증한 루프를 Unity로 옮긴다.

포함 범위:

- Android 세로 화면 프로젝트
- 터치 이동
- 자동 공격
- 몬스터/보스
- 레벨업 선택 UI
- 영구 성장 저장
- 기본 사운드
- 테스트 빌드

완료 기준:

- Android 기기 또는 에뮬레이터에서 실행된다.
- APK 또는 AAB 빌드가 가능하다.
- 5분 이상 플레이해도 치명적 오류가 없다.

### 3단계: 서비스 준비 버전

출시 가능성을 위해 최소 운영 요소를 붙인다.

포함 범위:

- 튜토리얼
- 설정 화면
- 개인정보처리방침 링크
- 광고 SDK 검토
- 크래시/로그 수집 검토
- 앱 아이콘
- 스토어 설명문
- 스크린샷
- 난이도/보상 밸런스

완료 기준:

- Google Play 내부 테스트 업로드 준비가 된다.
- 개인정보, 광고, 데이터 수집 항목을 설명할 수 있다.
- 폐쇄 테스트 피드백을 받을 수 있다.

### 4단계: 폐쇄 테스트

Google Play 개인 개발자 계정 기준으로 폐쇄 테스트 요건을 만족해야 한다.

현재 확인된 기준:

- 최소 12명 테스터
- 14일 연속 opt-in
- 이후 프로덕션 액세스 신청

정책은 변경될 수 있으므로 실제 제출 전 Google Play 공식 문서를 다시 확인한다.

### 5단계: 프로덕션 출시

폐쇄 테스트 결과를 반영한 뒤 출시한다.

출시 전 확인:

- 심각한 크래시 없음
- 첫 5분 플레이 루프 정상
- 광고/과금 정책 위반 없음
- 스토어 등록 정보 완성
- 개인정보처리방침 공개
- 앱 서명 및 빌드 타깃 기준 충족

## 다음 검증 질문

1. 60초 안에 성장감이 느껴지는가
2. 레벨업 선택지가 의미 있게 다른가
3. 실패해도 영구 강화 때문에 다시 하고 싶은가
4. 모바일 터치 조작이 피곤하지 않은가
5. 광고 보상 없이도 기본 진행이 막히지 않는가
