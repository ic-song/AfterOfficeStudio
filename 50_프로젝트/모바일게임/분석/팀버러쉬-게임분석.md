## 한 줄 요약

나무 베기라는 단순 행동을 짧은 반복 런, 자원 수집, 런 중 랜덤 성장, 영구 스킬 트리, 캐릭터/도끼 수집으로 확장한 증분형 로그라이트 게임이다.

## 핵심 구조

```text
런 시작
-> 거대한 나무를 베며 로그, 골드, 파워업 획득
-> 이동 또는 자동 수집으로 자원 회수
-> 레벨업마다 업그레이드 선택
-> 특수 로그와 시너지로 한 런의 폭발력 증가
-> 런 종료
-> 영구 스킬 트리, 도끼, 캐릭터 강화
-> 다음 런에서 더 빠르게 성장
```

Timber Rush의 중요한 점은 조작이 복잡하지 않다는 것이다. 플레이어가 배우는 규칙은 "베고, 줍고, 고르고, 강화한다" 정도로 단순하다. 대신 숫자 증가, 자동화, 특수 효과, 스킬 트리 확장으로 장기 몰입을 만든다.

## 플레이 감각

| 요소 | 분석 |
| --- | --- |
| 즉시 보상 | 나무를 칠 때 자원이 계속 튀어나와 행동과 보상이 거의 즉시 연결된다 |
| 짧은 런 | 한 번의 실패나 비효율이 부담스럽지 않고, 반복 진입이 쉽다 |
| 선택 피로 | 업그레이드가 자주 나오면 선택 자체가 또 다른 클릭 루프가 된다 |
| 자동화 | 초반에는 직접 움직임과 수집이 중요하지만, 성장 후에는 자동 수집/보조 캐릭터가 비중을 높인다 |
| 폭발 구간 | 특수 로그, 자석, 시간 정지, 번개 같은 효과가 숫자 폭증 순간을 만든다 |

## 성장 시스템

### 런 안 성장

- 레벨업 시 업그레이드 선택
- 특정 런에서만 강해지는 로그라이트식 변주
- 좋은 조합이 나오면 한 런의 효율이 크게 상승
- 운이 나쁜 런도 다음 영구 성장 재료를 얻는 과정으로 기능

### 런 밖 성장

- 로그를 사용한 대형 스킬 트리
- 자동화, 특수 로그, 포션, 시너지 해금
- 도끼와 캐릭터 해금/강화
- 반복 플레이의 피로를 줄이기 위해 점진적으로 직접 조작 부담을 줄임

이 구조는 "이번 판의 재미"와 "계정 전체의 성장"을 분리한다. 모바일 게임으로 가져올 때도 이 분리가 중요하다. 한 판은 가볍게 끝나야 하지만, 다음 판을 켜야 할 이유는 영구 성장에 있어야 한다.

## 잘 작동하는 이유

1. 주제가 단순하다.

   나무를 벤다는 목표가 명확하다. 세계관 설명이나 전투 규칙을 길게 읽지 않아도 플레이어가 무엇을 해야 하는지 바로 이해한다.

2. 보상 밀도가 높다.

   자원, 골드, 파워업, 레벨업 선택, 스킬 트리 해금이 촘촘하게 배치되어 있다. 플레이어는 거의 매 순간 "뭔가 얻고 있다"고 느낀다.

3. 조작과 방치의 중간 지점을 잡는다.

   완전 방치형이면 초반 손맛이 약하고, 완전 액션이면 증분형 유저에게 피곤하다. Timber Rush는 직접 이동과 자동화를 섞어서 두 감각을 모두 노린다.

4. 숫자 성장의 명분이 직관적이다.

   더 잘 베고, 더 많이 줍고, 더 빨리 성장한다. 공격력, 수집력, 자동화가 모두 핵심 목표에 직접 연결된다.

## 약점과 리스크

| 리스크 | 설명 | 우리 게임 적용 시 주의점 |
| --- | --- | --- |
| 반복 피로 | 핵심 행동이 단순해서 중후반에는 같은 행동의 반복으로 느껴질 수 있다 | 스테이지 목표, 적 패턴, 보스, 이벤트 로그 등 변주가 필요하다 |
| 초반 수집 스트레스 | Reddit 피드백에는 자원 수집을 위해 이동해야 하는 구조가 답답하다는 의견이 있었다 | 모바일에서는 드래그/이동 수집보다 자동 회수 또는 넓은 흡입 범위가 더 안전하다 |
| UI 혼잡 | 스킬 트리와 업그레이드가 많아지면 무엇을 살 수 있는지 파악이 어려워진다 | 구매 가능 강조, 추천 강화, 필터, 간단한 툴팁이 필요하다 |
| 설명 부족 | 도끼 해금/구매처럼 시스템 구분이 불명확하면 플레이어가 손해를 봤다고 느낄 수 있다 | 해금, 장착, 구매, 강화 상태를 명확히 분리해야 한다 |
| 후반 밸런스 폭주 | 공식 업데이트 글에서도 후반이 더 쉽게 "깨지는" 방향의 언급이 있다 | 숫자 폭발은 재미지만, 목표 상실을 막는 장기 목표가 필요하다 |

## 우리 모바일 게임에 주는 힌트

현재 모바일 게임 방향이 캐주얼 판타지 디펜스라면, Timber Rush에서 바로 가져올 수 있는 것은 "나무 베기 테마"가 아니라 성장 루프다.

| Timber Rush 요소 | 모바일 판타지 디펜스 적용 |
| --- | --- |
| 거대한 나무 하나 | 성문, 마법진, 수정탑, 도서관 같은 단일 방어 목표 |
| 로그/골드 수집 | 마나, 골드, 룬 조각, 몬스터 전리품 |
| 도끼/캐릭터 | 마법사, 지팡이, 정령, 주문서 |
| 특수 로그 | 특수 몬스터, 마나 결정, 룬 상자, 필드 이벤트 |
| 스킬 트리 | 영구 주문 강화, 자동 시전, 보상 증가, 생존 강화 |
| 짧은 런 | 1~3분 웨이브 방어 |

## MVP 반영 제안

처음부터 거대한 스킬 트리와 수십 개의 장비를 만들 필요는 없다. Timber Rush식 재미를 검증하려면 아래 정도가 적당하다.

| 항목 | MVP 범위 |
| --- | --- |
| 런 길이 | 60~120초 |
| 기본 목표 | 단일 방어 목표를 지키며 웨이브 버티기 |
| 런 중 선택 | 레벨업마다 3개 주문/버프 중 1개 선택 |
| 영구 성장 | 공격력, 자원 획득량, 자동 시전, 시작 보너스 4계열 |
| 특수 이벤트 | 자원 폭발, 화면 전체 공격, 시간 감속 중 1~2개 |
| 자동화 | 초반에는 약하게, 성장 후 자동 수집/자동 시전 강화 |
| 수익화 가능성 | 광고 보상, 스타터 패키지, 광고 제거는 가능성으로만 검토. 아직 검증된 수익 모델은 아님 |

## 설계 결론

Timber Rush의 강점은 "소재"보다 "성장 압축"에 있다. 플레이어가 짧은 시간 안에 자원을 얻고, 선택하고, 강해지고, 다음 판에서 더 빨라지는 흐름을 매우 노골적으로 보여준다.

우리 게임이 이 장점을 가져오려면 전투를 복잡하게 만들기보다 다음 세 가지를 먼저 검증해야 한다.

1. 1분 안에 플레이어가 강해졌다고 느끼는가
2. 실패해도 다음 판을 누를 이유가 생기는가
3. 자동화가 열릴수록 피로가 줄고 성장감이 커지는가

이 세 가지가 확인되면, 이후에 캐릭터, 장비, 보스, 스테이지, 수익화 요소를 붙여도 늦지 않다.
