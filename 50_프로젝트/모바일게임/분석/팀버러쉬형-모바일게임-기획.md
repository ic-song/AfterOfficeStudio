## 한 줄 요약

플레이어가 숲의 마법 수정을 지키며 몰려오는 몬스터를 막고, 매 런마다 주문을 선택해 강해진 뒤, 획득한 룬으로 영구 성장을 쌓아 다음 런을 더 빠르게 돌파하는 모바일 증분형 로그라이트 게임이다.

## 핵심 구조

```text
런 시작
-> 마법 수정 주변에서 몬스터 웨이브 방어
-> 자동 주문으로 몬스터 처치
-> 룬 조각, 골드, 경험치 획득
-> 레벨업마다 주문/버프 3개 중 1개 선택
-> 특수 몬스터 또는 룬 상자로 폭발 보상 획득
-> 60~120초 생존 또는 보스 처치
-> 룬으로 영구 성장 강화
-> 다음 런에서 더 빠르고 강하게 진행
```

Timber Rush가 "나무를 베고 자원을 회수하는 성장 압축"을 사용한다면, 이 기획은 "마법 수정을 지키고 룬을 회수하는 성장 압축"으로 바꾼다. 조작은 단순하게 두고, 선택과 성장의 빈도를 높여 짧은 모바일 세션에 맞춘다.

## 플레이 감각

| 요소 | 기획 방향 |
| --- | --- |
| 즉시 보상 | 몬스터 처치 시 룬 조각과 골드가 바로 떨어지고, 가까이 가면 자동 흡수된다 |
| 짧은 런 | 1~2분 안에 승패 또는 보스 결과가 나온다 |
| 선택 재미 | 레벨업마다 주문 강화, 신규 주문, 자원 보너스 중 하나를 고른다 |
| 자동화 | 기본 공격은 자동이고, 플레이어는 위치 조정과 선택에 집중한다 |
| 폭발 구간 | 룬 상자, 정예 몬스터, 화면 전체 주문으로 숫자가 크게 터지는 순간을 만든다 |
| 재도전 이유 | 실패해도 룬을 얻어 영구 강화가 가능하다 |

## 성장 시스템

### 런 안 성장

- 경험치를 얻으면 레벨업한다.
- 레벨업마다 3개 선택지 중 하나를 고른다.
- 선택지는 `주문 추가`, `주문 강화`, `자원 보너스`, `생존 보너스`로 나눈다.
- 같은 주문을 여러 번 고르면 효과가 눈에 띄게 강해진다.
- 런 안 성장은 해당 런이 끝나면 사라진다.

예시 선택지:

| 선택지 | 효과 |
| --- | --- |
| 불꽃 화살 | 가장 가까운 적에게 자동 발사 |
| 번개 사슬 | 적 사이를 튕기는 광역 피해 |
| 서리 파동 | 주변 적을 느리게 함 |
| 룬 흡수 | 룬 조각 흡수 범위 증가 |
| 마력 과부하 | 모든 주문 피해 증가 |
| 수호막 | 수정 또는 플레이어 체력 보호 |

### 런 밖 성장

- 런에서 얻은 룬으로 영구 스킬 트리를 강화한다.
- 영구 성장은 다음 런의 속도와 편의성을 높인다.
- 초반에는 공격력과 보상량을, 중반 이후에는 자동화와 시작 보너스를 열어준다.

영구 성장 계열:

| 계열 | 역할 |
| --- | --- |
| 공격 | 주문 피해, 발사 속도, 치명타 |
| 수집 | 룬 흡수 범위, 골드 획득량, 보상 증가 |
| 생존 | 수정 체력, 보호막, 회복 |
| 자동화 | 자동 룬 회수, 시작 주문, 자동 선택 보조 |

## 잘 작동하게 만들 조건

1. 첫 30초 안에 강해지는 느낌이 있어야 한다.

   첫 런에서도 최소 2~3번의 레벨업 선택이 발생해야 한다. 첫 플레이가 느리면 증분형 게임의 장점이 보이기 전에 이탈할 수 있다.

2. 수집 피로를 줄여야 한다.

   모바일에서는 작은 자원을 직접 따라다니는 행동이 금방 피곤해진다. 기본 흡수 범위를 넓게 주고, 영구 성장으로 자동 회수를 빠르게 열어야 한다.

3. 선택지는 적지만 차이가 커야 한다.

   초반 선택지가 너무 많으면 읽는 게임이 된다. MVP에서는 주문 6개, 보조 버프 6개 정도로 시작하고, 시각 효과와 역할 차이를 크게 만든다.

4. 실패도 손해처럼 느껴지면 안 된다.

   실패 후에도 룬, 골드, 해금 진행도가 남아야 한다. "졌지만 다음 판은 더 강하다"가 핵심이다.

## 약점과 리스크

| 리스크 | 설명 | 대응 방향 |
| --- | --- | --- |
| 단순 반복 | 자동 공격만 보면 금방 지루해질 수 있다 | 정예 몬스터, 보스, 룬 상자, 웨이브 변주를 넣는다 |
| 광고 피로 | 보상형 광고가 성장 루프를 끊을 수 있다 | 광고는 선택형 보상으로만 두고 기본 진행을 막지 않는다 |
| 숫자 인플레이션 | 증분형 성장으로 수치가 빠르게 커질 수 있다 | MVP에서는 수치 단위를 작게 유지하고, 성장 곡선을 짧게 검증한다 |
| 화면 혼잡 | 몬스터, 룬, 주문 효과가 겹치면 모바일 화면이 복잡해진다 | 주문 효과 색상과 크기를 제한하고, 줍는 자원은 빠르게 흡수한다 |
| 차별성 부족 | Vampire Survivors류 자동 전투로만 보일 수 있다 | "수정 방어 + 룬 성장 + 자동화 해금"을 전면에 둔다 |

## MVP 반영 제안

| 항목 | MVP 범위 |
| --- | --- |
| 스테이지 | 숲 1개 배경 |
| 런 길이 | 60초 생존 또는 미니 보스 처치 |
| 플레이어 | 마법사 1명 |
| 방어 목표 | 마법 수정 1개 |
| 몬스터 | 일반 몬스터 2종, 정예 1종, 보스 1종 |
| 주문 | 불꽃 화살, 번개 사슬, 서리 파동, 독구름, 룬 폭발, 수호막 |
| 런 중 선택 | 레벨업마다 3개 중 1개 |
| 영구 성장 | 공격, 수집, 생존, 자동화 4계열 |
| 보상 | 룬, 골드, 보상형 광고 2배 옵션 |
| 실패 처리 | 획득한 룬 일부 보존, 광고 시 추가 보상 |

## 초기 플레이 흐름

```text
1. 플레이 버튼 선택
2. 숲 스테이지 진입
3. 자동 불꽃 화살로 첫 몬스터 처치
4. 룬 조각 흡수
5. 레벨업 선택지 3개 등장
6. 주문 조합으로 60초 방어
7. 보스 또는 최종 웨이브 등장
8. 성공/실패 결과 화면
9. 룬으로 영구 강화
10. 다음 런 시작
```

## 수익화 가능성

이 내용은 가능성 검토이며 확정된 사업 모델이 아니다.

| 지점 | 방식 |
| --- | --- |
| 런 종료 후 | 광고 시 룬/골드 2배 |
| 실패 후 | 광고 시 추가 룬 또는 1회 부활 |
| 일일 보상 | 광고 시 추가 상자 |
| 편의 구매 | 광고 제거 |
| 초반 패키지 | 시작 룬, 스킨, 광고 제거 묶음 |

MVP에서는 수익화보다 반복 플레이 지표를 먼저 확인해야 한다. `다음 런 시작률`, `첫 5분 유지`, `영구 강화 구매 빈도`가 확인되기 전에는 과금 구조를 크게 만들지 않는다.

## 설계 결론

이 게임은 Timber Rush의 표면 소재를 따라 하는 게임이 아니라, Timber Rush의 성장 구조를 모바일 판타지 디펜스로 번역한 게임이다.

핵심 검증 질문은 세 가지다.

1. 60초 안에 플레이어가 강해졌다고 느끼는가
2. 실패 후에도 다음 런을 누르고 싶은가
3. 영구 성장으로 반복 피로가 줄어드는가

이 세 가지가 확인되면 다음 단계에서 캐릭터 수집, 장비, 스테이지, 장기 퀘스트, 시즌 이벤트를 붙인다. 확인 전에는 콘텐츠를 늘리기보다 1런의 보상 밀도와 성장 속도를 먼저 다듬는다.
