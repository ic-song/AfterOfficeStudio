# PROJECT.md

## Idea Name

Mass-Production Mobile Game for Android

## Purpose

Explore a repeatable production framework for small Android mobile games.

This is an idea record, not an active project. The goal is to clarify what kind of "mass-production" mobile game could be built, then decide whether it should be promoted to a project.

## Target Users

Unconfirmed.

Candidate target users:

- Android users who play lightweight casual games in short sessions.
- Solo developers or small teams who need a reusable game production pattern.
- Internal After Office Studio operators who want to test repeatable game production.

## Problem

Unconfirmed.

Candidate problems:

- Small mobile game ideas are easy to start but hard to package into a repeatable production process.
- Development scope can expand before the core loop, monetization possibility, and release path are validated.
- Android release, testing, analytics, and asset pipelines may become repeated manual work unless standardized.

## Initial Idea Review Scope

The first review scope is:

- Define the game genre candidates.
- Define the target user and play session.
- Define the MVP game loop.
- Review Android technical stack options.
- Decide whether this idea should be promoted to an active project.

## Business Model Possibilities

Speculation only. No business model is confirmed.

Possible models to review later:

- Ads-based casual game revenue.
- In-app purchase for simple progression or cosmetics.
- Reusable internal template for producing multiple small games.
- Portfolio or learning project without direct revenue.
