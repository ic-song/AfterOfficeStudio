# Token Usage Rules 한국어 검토본

## 목적

이 문서는 After Office Studio에서 AI Agent가 토큰을 아끼면서도 정확하게 작업하기 위한 기준을 정의합니다.

목표는 불필요한 전체 문서 읽기, 반복 설명, 중복 Agent 전달, 광범위한 문서 스캔을 줄이면서 정확성과 검토 가능성을 유지하는 것입니다.

## 핵심 원칙

```text
필요한 만큼만 읽는다.
필요한 만큼만 쓴다.
위험도가 정당화될 때만 역할 분담을 늘린다.
```

## 문서 읽기 순서

1. 먼저 진입점 또는 인덱스를 확인합니다.
2. 검색으로 관련 문서와 섹션을 찾습니다.
3. 필요한 파일이나 섹션만 읽습니다.
4. 전체 맥락이 필요할 때만 전체 파일을 읽습니다.
5. 변경 가능성이 없는 같은 내용을 반복해서 읽지 않습니다.

추천 진입점:

```text
AGENTS.md
01_rule/ai/document-index.md
01_rule/agents/core/role-map.md
```

## 1뎁스 폴더 인덱스

실제 작업에 사용하는 최상위 폴더에는 `INDEX.md`를 둡니다.

이 인덱스 파일은 토큰 절약을 위한 AI 라우팅 보조 문서입니다. 파일명이 한국어이더라도 본문은 영어로 작성하고 유지합니다.

`01_rule/` 밖의 AI 보조 인덱스에는 `.ko.md` 대응본을 만들지 않습니다. `01_rule/` 아래의 규칙 문서는 예외이며 필수 한국어 검토본을 유지합니다.

목적:

- 넓은 폴더 스캔 없이 작업 위치를 찾습니다.
- 현재 내용과 중요한 하위 문서를 보여줍니다.
- 다음에 열어야 할 파일을 Agent가 빠르게 선택하게 합니다.

기본 확인 순서:

```text
1. README.md
2. 01_rule/ai/document-index.md
3. {최상위폴더}/INDEX.md
4. 작업에 필요한 특정 문서만 확인
```

`INDEX.md`로 필요한 문서를 찾을 수 있으면 최상위 폴더 아래의 모든 파일을 읽지 않습니다.

## Agent 활성화 기준

작업을 안전하게 처리할 수 있는 최소 역할만 사용합니다.

```text
단순 질문                  -> 직접 답변
단일 규칙/문서 수정         -> Karma 기준
문서 구조 질문              -> Sona 기준
큰 구조 변경                -> 필요 시 Shen + Sona + Karma + Ryze
인코딩 위험                 -> Ryze
Git 실행                    -> Ekko
```

Ryze는 모든 문서 변경마다 호출하지 않고, 인코딩 위험이 있을 때 활성화합니다.

## 보고 기준

중간 보고는 짧게 유지하고 다음에 집중합니다.

- 무엇을 확인했는가
- 무엇을 바꿀 것인가
- 위험이나 확인 필요가 있는가

최종 보고는 다음에 집중합니다.

- 변경 파일
- 핵심 변경
- 검증 결과
- 남은 위험 또는 후속 작업

## 동기화 최적화

- 변경된 counterpart pair만 동기화합니다.
- 작은 변경을 위해 모든 Markdown 파일을 스캔하거나 다시 쓰지 않습니다.
- 큰 파일을 열기 전에 `rg` 또는 문서 인덱스를 사용합니다.
- 작은 수정은 변경 파일 검사부터 수행합니다.
- 전체 Markdown 검사는 high-risk sync, 복구, 릴리스, 커밋 준비 시에만 사용합니다.

## 인코딩 검사

단순 수정은 변경 파일 UTF-8 검사로 충분합니다.

다음 경우 Ryze를 활성화합니다.

- 한국어 문서 변경량이 큼
- mojibake 또는 깨진 텍스트가 의심됨
- 사용자가 인코딩 검토를 요청함
- 문서 복구가 필요함
- high-risk sync 작업이 rule-related Markdown에 닿음
